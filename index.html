Бинарный поиск

Алгоритм работает только на отсортированном массиве.

log2n — это степень, в которую нужно возвести 2, чтобы получить n

log₂50 → между 25=32 и 26=64 → ≈5.6 (6 шагов)

log₂200 → между 27=128 и 28=256 → ≈7.6 (8 шагов)

Правило "делим пополам":

Для бинарного поиска log₂n — это сколько раз нужно разделить n на 2, чтобы получить 1.

Пример для n=100:
        100 / 2 = 50
        50 / 2 = 25
        25 / 2 = 12.5
        12.5 / 2 = 6.25
        6.25 / 2 = 3.125
        3.125 / 2 = 1.5625
        1.5625 / 2 = 0.78125 (меньше 1, останавливаемся)
        Итого: 7 шагов
